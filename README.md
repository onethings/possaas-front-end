技術棧要求
框架: React 18+ (Vite)

狀態管理: 建議使用 React Query (TanStack Query) 來處理後端頻繁的庫存與訂單更新，並配合快取機制。

樣式: Tailwind CSS。

API Client: Axios (需設定 withCredentials 與攔截器)。

錯誤代碼定義 (Error Codes)

404: 找不到資源（例如租戶資料遺失）。

### 2. 多頁籤（Tab）內嵌管理介面
右側主工作區頂端採用 **分頁 (Tab) 內嵌設計**，需實現以下高級交互特點：
* **多頁籤並存**：用戶從左側導航點選功能時，主工作區頂部以 Tab 形式疊加（例如同時存在「商品列表」、「銷售匯總」與「按商品銷售報告」）。
* **快速切換與狀態保留**：允許用戶在不同分頁間自由點擊切換。**各分頁內的操作進度、輸入內容或未提交的查詢結果均需在內存中予以保留**，切換時不可重新掛載（Mount）或丟失數據。
* **獨立關閉**：每個頁籤右側均附有 `×` 關閉按鈕，點擊後銷毀該組件內存。

你可以上傳兩種不同用途的商標圖檔：

電子郵件收據已傳送：用於發送給客戶的電子收據商標。

收據列印：用於實體印表機列印出來的紙本收據商標。

2. 文字欄位
頁首：可輸入顯示在收據上方的文字（字數限制 0 / 500 字）。

頁尾：可輸入顯示在收據下方的文字（如：謝謝惠顧、退換貨說明等，字數限制 0 / 500 字）。

3. 開關選項（功能設定）
顯示客戶資訊：控制是否在收據上顯示客戶的相關資料（目前為關閉狀態）。

顯示註釋：控制是否在收據上顯示備註或註釋訊息（目前為關閉狀態）。

4. 語言與儲存
收據語言：可透過下拉選單調整收據的語系（目前設定為「中文（繁體）」）。

底部按鈕：提供「取消」與「儲存」功能。

「商店」分頁（商店和 POS 設置）中，主要包含以下內容和功能：

1. 新增功能
「+ 新增商店」按鈕：位於頁面左上方，點擊後可以用來建立新的商店據點。

2. 商店列表資訊欄位
右側的表格展示了目前已有的商店清單，包含以下三個資訊欄位：

名稱：商店的區別名稱。

地址：商店的實體地址（目前畫面中此欄位為空）。

POS 的數目：該商店目前綁定的 POS 設備數量。

3. 現有商店資料
列表中目前有一家名為 「POL」 的商店，其顯示的 POS 的數目為 2。

「POS 設備」這個頁面中，主要包含以下內容和功能：

1. POS 設備列表
頁面中央的表格列出了目前系統中現有的 POS 設備，包含以下欄位：

+ 新增 POS 按鈕：位於列表左上方（綠色按鈕），點擊此按鈕可以讓您在系統中添加新的 POS 設備。
